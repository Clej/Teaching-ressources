TP1¶

L'objectif de ce TP est, dans un premier temps, de se familiariser avec la régression linéaire (RL) dans un cadre prédictif. Nous allons utiliser le jeu de données "Boston Houssing Prices" présenté en cours.

Dans un second temps, nous aborderons une des limites de la régression linéaire: la dimension i.e. ce qui se passe quand le nombre de variables $X_1, \dots, X_d$ devient très grand, ce qui nous amènera au second cours.

Apprentissage des paramètres $w_{1}, \dots, w_{d}$¶

Estimateur "brute"¶

On va estimer les paramtères du modèle en utilisant les opérateurs matriciels de NumPy, puis on utilisera et comperera les résultats avec un estimateur de Scikit-learn.

1- Ecrire une fonction qui prédit la variable cible $y_{i}^{pred}$ à partir de paramtères et d'une observation $x_{i}$ donnés.

In [ ]:
def pred_y(x, beta):
    """prediction de y selon x, beta"""
    pred = x @ beta
    return pred

2- Ecrire une fonction qui prend en entrée:

  • un ensemble d'observations de la variable cible $y_{1}, \dots y_{i}, \dots$
  • les prédictions associées $y_{1}^{pred} = \sum_j w_{j} x_{1j}, \dots y_{i}^{pred} = \sum_{j} w_{j} x_{ij}, \dots$

et retourne en sortie l'erreur quadratique moyenne.

In [ ]:
def err_quad(y_obs, y_pred):
    """erreur quadratique moyenne"""
    n = len(y_obs)
    e_i = y_obs - y_pred
    err_quad = np.square(e_i)
    res = 0.5 * (1/n) * np.sum(err_quad)
    return res

3- Ecrire une fonction estime les paramètres $w_{1}, \dots, w_{d}$, à partir des observations $\{ y_i, x_i \}_{i \leq n}$ de façon à minimiser l'erreur quadratique moyenne.

In [ ]:
def learn_w(X, Y):
    """solution des moindres carrés linéaires"""
    w_hat = np.linalg.inv(X.T @ X) @ X.T @ Y
    return w_hat

4- Ecrire une fonction qui séparer les observations du jeu de données aléatoirement en un ensemble d'entraînement et un ensemble de test.

In [ ]:
np.random.seed(3011)
def split_train_test(X, Y, ratio=0.7):
    """train/test split"""
    n = len(Y)
    n_tr = int(ratio * n)
    idx_train = np.random.randint(low=0, high=n-1, size=n_tr-1) # indices python commence à 0.
    idx_test = np.setdiff1d(ar1=np.arange(0, n-1), ar2=idx_train)
    idx_train, idx_test = list(idx_train), list(idx_test)
    Y_train, X_train = Y[idx_train], X[idx_train, :]
    Y_test, X_test = Y[idx_test], X[idx_test, :]
    # centrage des données:
    X_mean, y_mean = X_train.mean(axis=0), Y_train.mean()
    X_train -= X_mean
    X_test -= X_mean
    Y_train -= y_mean
    Y_test -= y_mean 
    return X_train, X_test, Y_train, Y_test

Pour avoir une meilleure estimation de l'erreur de généralisation qu'avec une seule séparation entraînement/test, on répètre la procédure plusieurs fois puis on calcul la moyenne des erreurs de généralisations. Cette procédure s'appelle la "validation croisée".

6- Ecrire une fonction qui réalise la validation croisée, puis donnez une estimation de l'erreur de généralisation.

In [ ]:
def cv_rl(X, y, n_cv=10):
    err_cv = np.empty(shape=n_cv)
    for k in range(n_cv):
        X_tr, X_ts, y_tr, y_ts = split_train_test(X=X, Y=y, ratio=0.80)
        w_hat_k = learn_w(Y=y_tr, X=X_tr)
        pred = pred_y(X_ts, w_hat_k)
        err_cv[k] = err_quad(y_obs=y_ts, y_pred=pred)

    return err_cv

Partie II¶

Etude de l'effet dimension sur l'apprentissage des paramètres¶

Jusqu'ici on a considéré un nombre de variables faible et un nombre d'observations "important". Souvent, et c'est de plus en plus fréquent avec l'ère "big data", le nombre de variables $d$ est grand devant le nombre d'obersvations $n$. Cela pose des souci de calcul, $\hat{w}$ est très couteûx à calculer, d'incertitude de prédiction et également d'analyse du modèle appris. Si les données comportent, par exemple, plus de $d=500$ variables, il y a de grande chance que

  • certaines soient totalement décorrélées de la variable cible,
  • certaines sont fortement corrélées à d'autres variables predictives.

Dans le premier cas, on a un souci d'interprétabilité (utilisation de variables à tort), on aimerait supprimer ces variables "inutiles". Dans le deuxième cas, la prédiction du modèle appris deviendra plus incertaine, puisque le coefficient d'une variable corrélée à d'autre prédicteurs ne sera jamais nul, on ne sait pas vraiment si il a un petit effet sur la prédiction ou bien si cela dû à l'aléa présent dans les données.

On appelle ce phénomène la "malédiction de la dimensionalité".

Regardons cela empiriquement sur des données artificielles.

In [ ]:
err_d = np.empty(d)
std_d = np.empty(d)

for j in range(d):
    rl_hd = LinearRegression()
    if j < d_util:
        # erreur de généralisation du modèle appris avec des variables utiles pour prédire y
        scores = cross_val_score(rl_hd, X[:, idx_util[:(j+1)]], y,
                                scoring="neg_mean_squared_error", cv=5)
    else:
        # erreur de généralisation du modèle appris avec toutes les variables utiles 
        # et d'autes inutiles pour prédire y
        scores = cross_val_score(rl_hd, X[:, idx_util + idx_nonutil[:(j+1-d_util)]], y,
                                scoring="neg_mean_squared_error", cv=5)
    err_d[j] = -scores.mean()
    std_d[j] = scores.std()

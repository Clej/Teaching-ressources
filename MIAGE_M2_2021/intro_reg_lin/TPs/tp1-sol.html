1- Ecrire une fonction qui prdit la variable cible $y_{i}^{pred}$ partir de paramtres et d'une observation $x_{i}$ donns.

In [ ]:
def pred_y(x, beta):
    """prediction de y selon x, beta"""
    pred = x @ beta
    return pred

2- Ecrire une fonction qui prend en entre:

  • un ensemble d'observations de la variable cible $y_{1}, \dots y_{i}, \dots$
  • les prdictions associes $y_{1}^{pred} = \sum_j w_{j} x_{1j}, \dots y_{i}^{pred} = \sum_{j} w_{j} x_{ij}, \dots$

et retourne en sortie l'erreur quadratique moyenne.

In [ ]:
def err_quad(y_obs, y_pred):
    """erreur quadratique moyenne"""
    n = len(y_obs)
    e_i = y_obs - y_pred
    err_quad = np.square(e_i)
    res = 0.5 * (1/n) * np.sum(err_quad)
    return res

3- Ecrire une fonction estime les paramtres $w_{1}, \dots, w_{d}$, partir des observations $\{ y_i, x_i \}_{i \leq n}$ de faon minimiser l'erreur quadratique moyenne.

In [ ]:
def learn_w(X, Y):
    """solution des moindres carrs linaires"""
    w_hat = np.linalg.inv(X.T @ X) @ X.T @ Y
    return w_hat

4- Ecrire une fonction qui sparer les observations du jeu de donnes alatoirement en un ensemble d'entranement et un ensemble de test.

In [ ]:
np.random.seed(3011)
def split_train_test(X, Y, ratio=0.7):
    """train/test split"""
    n = len(Y)
    n_tr = int(ratio * n)
    idx_train = np.random.randint(low=0, high=n-1, size=n_tr-1) # indices python commence  0.
    idx_test = np.setdiff1d(ar1=np.arange(0, n-1), ar2=idx_train)
    idx_train, idx_test = list(idx_train), list(idx_test)
    Y_train, X_train = Y[idx_train], X[idx_train, :]
    Y_test, X_test = Y[idx_test], X[idx_test, :]
    # centrage des donnes:
    X_mean, y_mean = X_train.mean(axis=0), Y_train.mean()
    X_train -= X_mean
    X_test -= X_mean
    Y_train -= y_mean
    Y_test -= y_mean 
    return X_train, X_test, Y_train, Y_test

def cv_rl(X, y, n_cv=10):
    err_cv = np.empty(shape=n_cv)
    for k in range(n_cv):
        X_tr, X_ts, y_tr, y_ts = split_train_test(X=X, Y=y, ratio=0.80)
        w_hat_k = learn_w(Y=y_tr, X=X_tr)
        pred = pred_y(X_ts, w_hat_k)
        err_cv[k] = err_quad(y_obs=y_ts, y_pred=pred)

    return err_cv

Partie II¶

Etude de l'effet dimension sur l'apprentissage des paramètres¶

Jusqu'ici on a considr un nombre de variables faible et un nombre d'observations "important". Souvent, et c'est de plus en plus frquent avec l're "big data", le nombre de variables $d$ est grand devant le nombre d'obersvations $n$. Cela pose des souci de calcul, $\hat{w}$ est trs coutex calculer, d'incertitude de prdiction et galement d'analyse du modle appris. Si les donnes comportent, par exemple, plus de $d=500$ variables, il y a de grande chance que

  • certaines soient totalement dcorrles de la variable cible,
  • certaines sont fortement corrles d'autres variables predictives.

Dans le premier cas, on a un souci d'interprtabilit (utilisation de variables tort), on aimerait supprimer ces variables "inutiles". Dans le deuxime cas, la prdiction du modle appris deviendra plus incertaine, puisque le coefficient d'une variable corrle d'autre prdicteurs ne sera jamais nul, on ne sait pas vraiment si il a un petit effet sur la prdiction ou bien si cela d l'ala prsent dans les donnes.

On appelle ce phnomne la "maldiction de la dimensionalit".

Regardons cela empiriquement sur des donnes artificielles.

    if j < d_util:
        # erreur de gnralisation du modle appris avec des variables utiles pour prdire y
        scores = cross_val_score(rl_hd, X[:, idx_util[:(j+1)]], y,

    std_d[j] = scores.std()

1- Ecrire une fonction qui prdit la variable cible $y_{i}^{pred}$ partir de paramtres et d'une observation $x_{i}$ donns.

In [ ]:
def pred_y(x, beta):
    """prediction de y selon x, beta"""
    pred = x @ beta
    return pred

2- Ecrire une fonction qui prend en entre:

  • un ensemble d'observations de la variable cible $y_{1}, \dots y_{i}, \dots$
  • les prdictions associes $y_{1}^{pred} = \sum_j w_{j} x_{1j}, \dots y_{i}^{pred} = \sum_{j} w_{j} x_{ij}, \dots$

et retourne en sortie l'erreur quadratique moyenne.

In [ ]:
def err_quad(y_obs, y_pred):
    """erreur quadratique moyenne"""
    n = len(y_obs)
    e_i = y_obs - y_pred
    err_quad = np.square(e_i)
    res = 0.5 * (1/n) * np.sum(err_quad)
    return res

3-1 Ecrire une fonction estime les paramtres $w_{1}, \dots, w_{d}$, partir des observations $\{ y_i, x_i \}_{i \leq n}$ de faon minimiser l'erreur quadratique moyenne.

In [ ]:
def learn_w(X, Y):
    """solution des moindres carrs linaires ordinaires"""
    w_hat = np.linalg.inv(X.T @ X) @ X.T @ Y
    return w_hat

3-2 Ecrire une fonction qui estime les paramtres selon la rgularisation ridge de paramtre $\lambda$.

In [ ]:
def learn_w_ridge(X, Y, lam=0.001):
    """solution ridge"""
    d = X.shape[1]
    w_hat = np.linalg.inv(X.T @ X + lam * np.identity(d)) @ X.T @ Y
    return w_hat

4- Ecrire une fonction qui sparer les observations du jeu de donnes alatoirement en un ensemble d'entranement et un ensemble de test.

In [ ]:
np.random.seed(3011)
def split_train_test(X, Y, ratio=0.7):
    """train/test split"""
    n = len(Y)
    n_tr = int(ratio * n)
    idx_train = np.random.randint(low=0, high=n-1, size=n_tr-1) # indices python commence  0.
    idx_test = np.setdiff1d(ar1=np.arange(0, n-1), ar2=idx_train)
    idx_train, idx_test = list(idx_train), list(idx_test)
    Y_train, X_train = Y[idx_train], X[idx_train, :]
    Y_test, X_test = Y[idx_test], X[idx_test, :]
    # centrage des donnes:
    X_mean, y_mean = X_train.mean(axis=0), Y_train.mean()
    X_train -= X_mean
    X_test -= X_mean
    Y_train -= y_mean
    Y_test -= y_mean 
    return X_train, X_test, Y_train, Y_test

Pour avoir une meilleure estimation de l'erreur de gnralisation qu'avec une seule sparation entranement/test, on rptre la procdure plusieurs fois puis on calcul la moyenne des erreurs de gnralisations. Cette procdure s'appelle la "validation croise".

6- Ecrire une fonction qui ralise la validation croise, puis donnez une estimation de l'erreur de gnralisation.

In [ ]:
def cv_rl(X, y, n_cv=10):
    err_cv = np.empty(shape=n_cv)
    for k in range(n_cv):
        X_tr, X_ts, y_tr, y_ts = split_train_test(X=X, Y=y, ratio=0.80)
        w_hat_k = learn_w(Y=y_tr, X=X_tr)
        pred = pred_y(X_ts, w_hat_k)
        err_cv[k] = err_quad(y_obs=y_ts, y_pred=pred)

    return err_cv

def cv_rl_ridge(X, y, lam, n_cv=10):

        pred = pred_y(X_ts, w_hat_k)
        err_cv[k] = err_quad(y_obs=y_ts, y_pred=pred)

    return err_cv

Choix du paramètre de régularissation $\lambda$¶

En pratique $\lambda$ est un paramtre que l'on ne peut pas apprendre, on le qualifie d'hyper-paramtre. Pour le slectionner on se sert de l'erreur de gnralisation:

  • Spcifier une ensemble de valeurs candidates pour $\lambda_{1} \dots \lambda_{k} \dots \lambda_{M}$, Puis pour chacune de ces valeurs: (disons $\lambda_{k}$)
    • entraner le modle sur un ensemble de donnes d'entranement avec $\lambda_{k}$, on obtient les paramtres appris $\hat{w}_{\lambda_{k}}$,
    • valuer l'erreur de gnralisation avec $\hat{w}_{\lambda_{k}}$ sur un ensemble de donnes de test,

Choisir la valeur de $\lambda$ teste qui donne la plus petite erreur de gnralisation.

Tout cela se fait en quelques lignes avec Scikit-learn ! (L'hyper-paramtre est appel 'alpha')

On compare la rgression linaire ordinaire (comme au-dessus) mais en utilisant uniquement scikit-learn.

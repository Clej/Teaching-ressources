Apprentissage des paramètres $w_{1}, \dots, w_{d}$¶

Estimateur "brute"¶

On va estimer les paramtères du modèle en utilisant les opérateurs matriciels de NumPy, puis on utilisera et comperera les résultats avec un estimateur de Scikit-learn.

On reprend le code du TP#1.

1- Ecrire une fonction qui prédit la variable cible $y_{i}^{pred}$ à partir de paramtères et d'une observation $x_{i}$ donnés.

In [ ]:
def pred_y(x, beta):
    """prediction de y selon x, beta"""
    pred = x @ beta
    return pred

2- Ecrire une fonction qui prend en entrée:

  • un ensemble d'observations de la variable cible $y_{1}, \dots y_{i}, \dots$
  • les prédictions associées $y_{1}^{pred} = \sum_j w_{j} x_{1j}, \dots y_{i}^{pred} = \sum_{j} w_{j} x_{ij}, \dots$

et retourne en sortie l'erreur quadratique moyenne.

In [ ]:
def err_quad(y_obs, y_pred):
    """erreur quadratique moyenne"""
    n = len(y_obs)
    e_i = y_obs - y_pred
    err_quad = np.square(e_i)
    res = 0.5 * (1/n) * np.sum(err_quad)
    return res

3-1 Ecrire une fonction estime les paramètres $w_{1}, \dots, w_{d}$, à partir des observations $\{ y_i, x_i \}_{i \leq n}$ de façon à minimiser l'erreur quadratique moyenne.

In [ ]:
def learn_w(X, Y):
    """solution des moindres carrés linéaires ordinaires"""
    w_hat = np.linalg.inv(X.T @ X) @ X.T @ Y
    return w_hat

3-2 Ecrire une fonction qui estime les paramètres selon la régularisation ridge de paramètre $\lambda$.

In [ ]:
def learn_w_ridge(X, Y, lam=0.001):
    """solution ridge"""
    d = X.shape[1]
    w_hat = np.linalg.inv(X.T @ X + lam * np.identity(d)) @ X.T @ Y
    return w_hat

4- Ecrire une fonction qui séparer les observations du jeu de données aléatoirement en un ensemble d'entraînement et un ensemble de test.

In [ ]:
np.random.seed(3011)
def split_train_test(X, Y, ratio=0.7):
    """train/test split"""
    n = len(Y)
    n_tr = int(ratio * n)
    idx_train = np.random.randint(low=0, high=n-1, size=n_tr-1) # indices python commence à 0.
    idx_test = np.setdiff1d(ar1=np.arange(0, n-1), ar2=idx_train)
    idx_train, idx_test = list(idx_train), list(idx_test)
    Y_train, X_train = Y[idx_train], X[idx_train, :]
    Y_test, X_test = Y[idx_test], X[idx_test, :]
    # centrage des données:
    X_mean, y_mean = X_train.mean(axis=0), Y_train.mean()
    X_train -= X_mean
    X_test -= X_mean
    Y_train -= y_mean
    Y_test -= y_mean 
    return X_train, X_test, Y_train, Y_test

Pour avoir une meilleure estimation de l'erreur de généralisation qu'avec une seule séparation entraînement/test, on répètre la procédure plusieurs fois puis on calcul la moyenne des erreurs de généralisations. Cette procédure s'appelle la "validation croisée".

6- Ecrire une fonction qui réalise la validation croisée, puis donnez une estimation de l'erreur de généralisation.

In [ ]:
def cv_rl(X, y, n_cv=10):
    err_cv = np.empty(shape=n_cv)
    for k in range(n_cv):
        X_tr, X_ts, y_tr, y_ts = split_train_test(X=X, Y=y, ratio=0.80)
        w_hat_k = learn_w(Y=y_tr, X=X_tr)
        pred = pred_y(X_ts, w_hat_k)
        err_cv[k] = err_quad(y_obs=y_ts, y_pred=pred)

    return err_cv
In [ ]:
def cv_rl_ridge(X, y, lam, n_cv=10):
    err_cv = np.empty(shape=n_cv)
    for k in range(n_cv):
        X_tr, X_ts, y_tr, y_ts = split_train_test(X=X, Y=y, ratio=0.80)
        w_hat_k = learn_w_ridge(Y=y_tr, X=X_tr, lam=lam)
        pred = pred_y(X_ts, w_hat_k)
        err_cv[k] = err_quad(y_obs=y_ts, y_pred=pred)

    return err_cv

Choix du paramètre de régularissation $\lambda$¶

En pratique $\lambda$ est un paramètre que l'on ne peut pas apprendre, on le qualifie d'hyper-paramètre. Pour le sélectionner on se sert de l'erreur de généralisation:

  • Spécifier une ensemble de valeurs candidates pour $\lambda_{1} \dots \lambda_{k} \dots \lambda_{M}$, Puis pour chacune de ces valeurs: (disons $\lambda_{k}$)
    • entraîner le modèle sur un ensemble de données d'entraînement avec $\lambda_{k}$, on obtient les paramètres appris $\hat{w}_{\lambda_{k}}$,
    • évaluer l'erreur de généralisation avec $\hat{w}_{\lambda_{k}}$ sur un ensemble de données de test,

Choisir la valeur de $\lambda$ testée qui donne la plus petite erreur de généralisation.

Tout cela se fait en quelques lignes avec Scikit-learn ! (L'hyper-paramètre est appelé 'alpha')

On compare à la régression linéaire ordinaire (comme au-dessus) mais en utilisant uniquement scikit-learn.

err_ridge_d = np.empty(d)

for j in range(d):
    rl_ridge = RidgeCV(alphas_cand)
    rl_ord = LinearRegression()
    if j < d_util:
        # erreur de généralisation du modèle appris avec des variables utiles pour prédire y
        scores_rl_ridge = cross_val_score(rl_ridge, X[:, idx_util[:(j+1)]], y,
                                scoring="neg_mean_squared_error", cv=5)
        scores_rl_ord = cross_val_score(rl_ord, X[:, idx_util[:(j+1)]], y,
                        scoring="neg_mean_squared_error", cv=5)
    else:
        # erreur de généralisation du modèle appris avec toutes les variables utiles 
        # et d'autes inutiles pour prédire y
        scores_rl_ridge = cross_val_score(rl_ridge, X[:, idx_util + idx_nonutil[:(j+1-d_util)]], y,
                                scoring="neg_mean_squared_error", cv=5)
        scores_rl_ord = cross_val_score(rl_ord, X[:, idx_util + idx_nonutil[:(j+1-d_util)]], y,
                                scoring="neg_mean_squared_error", cv=5)

    err_ridge_d[j] = -scores_rl_ridge.mean()
    std_ridge_d[j] = scores_rl_ridge.std()

    err_ord_d[j] = -scores_rl_ord.mean()
    std_ord_d[j] = scores_rl_ord.std()
